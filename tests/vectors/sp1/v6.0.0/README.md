# SP1 v6.0.0 Groth16 proof fixtures

This directory contains a real SP1 v6.0.0 Groth16 BN254 proof used by the
Rúnar witness-assisted verifier's end-to-end tests. The fixtures are
frozen at this specific SP1 version — any upgrade of the SP1 trusted setup
will invalidate them and require a full regeneration (including a fresh
`vk.json` extracted from the new Solidity verifier).

## Provenance

Generated by bsv-evm's `host-evm` prover, which runs a simplified EVM
balance-transfer guest program through the SP1 zkVM (core → compressed →
Groth16) and writes out the Groth16 proof artifacts. The source prover
binary is at `bsv-evm/prover/host-evm/src/main.rs`. The original artifacts
live under `bsv-evm/tests/sp1/` and were copied here verbatim.

## Files

| File | Bytes | Description |
|------|-------|-------------|
| `groth16_raw_proof.hex` | 324 | Raw gnark proof bytes (see layout below) |
| `groth16_encoded_proof.hex` | 352 | Solidity calldata layout (96-byte prefix + gnark A/B/C) |
| `groth16_public_inputs.txt` | — | 5 newline-separated decimal Fr scalars |
| `groth16_vkey_hash.hex` | 32 | SHA256 hash of the Groth16 circuit VK (identifies the circuit) |
| `public_values.hex` | 112 | Pre-hash guest public values (before SP1 hashes into public input 1) |
| `vk.json` | — | Machine-readable Groth16 verifying key (pre-negated β, γ, δ) |

## `groth16_raw_proof.hex` byte layout

This is gnark-crypto's **native `WriteRawTo` format** for a `groth16_bn254.Proof`,
as called by SP1's Go gnark-ffi wrapper (`NewSP1Groth16Proof` in
`sp1-recursion-gnark-ffi/go/sp1/utils.go`). The format is:

```
bytes 0..64     Ar (G1 uncompressed): x[32] || y[32], big-endian Fp
bytes 64..192   Bs (G2 uncompressed): x.A1[32] || x.A0[32] || y.A1[32] || y.A0[32]
                                       (imaginary-first Fp2, gnark native)
bytes 192..256  Krs (G1 uncompressed): x[32] || y[32]
bytes 256..260  Commitments slice length (uint32 big-endian) — zero for SP1
bytes 260..324  CommitmentPok (G1 uncompressed, 64 bytes) — all-zero for SP1
```

Note the Fp2 byte order: gnark stores G2 coordinates **imaginary part first**
(`x.A1 || x.A0 || y.A1 || y.A0`), so a swap is required to reach Rúnar's
`(x0=real, x1=imag)` convention. `bn254witness.ParseSP1RawProof` performs
this swap.

For SP1 v6 this proof has no Pedersen commitments (`Commitments` is empty)
and the `CommitmentPok` bytes are zeroed, so the "interesting" part of the
proof is the first 256 bytes. The remaining 68 bytes stay in the fixture
because they're part of gnark's raw format and a future SP1 upgrade might
start using commitments.

## `groth16_encoded_proof.hex` byte layout

This is the Solidity-ABI-encoded calldata form that SP1 uses to feed its
on-chain Solidity verifier. The layout is (from
`NewSP1Groth16Proof` in `sp1-recursion-gnark-ffi/go/sp1/utils.go`):

```
bytes  0.. 32  exit_code          (uint256 big-endian)
bytes 32.. 64  vk_root            (uint256 big-endian)
bytes 64.. 96  proof_nonce        (uint256 big-endian)
bytes 96..352  Gnark MarshalSolidity output — standard Groth16 Solidity ABI:
                 A.x (32) || A.y (32) || B.x.A1 (32) || B.x.A0 (32) ||
                 B.y.A1 (32) || B.y.A0 (32) || C.x (32) || C.y (32)
```

The Rúnar verifier does not use this file directly — we prefer
`groth16_raw_proof.hex` because it's closer to gnark's native format and
parsing it requires exactly one well-understood Fp2 swap. It's kept here
for cross-reference with EVM-side tooling.

## `groth16_public_inputs.txt` — the 5 SP1 public inputs

Each line is a decimal string for a BN254 Fr scalar. The SP1 Groth16 circuit
has exactly 5 public inputs:

| Index | Name | Meaning |
|------:|:-----|:--------|
| 0 | `vkey_hash` | SP1 program verifying key hash (Bn254 Fr, top 3 bits zeroed) |
| 1 | `committed_values_digest` | `SHA256(public_values)` with top 3 bits zeroed — the hash of the guest's `public_values.hex` |
| 2 | `exit_code` | Exit code of the guest (0 for success) |
| 3 | `vk_root` | Root of the recursion VK tree (constant per SP1 version) |
| 4 | `proof_nonce` | Proof nonce (unused in this fixture — 0) |

Indices 2 and 4 are zero for this fixture (success exit code, no nonce).

## `vk.json` schema

Machine-readable form of SP1 v6.0.0's Groth16 verifying key. All big
integers are stored as decimal strings to avoid JSON number precision
loss. The G2 points `betaNegG2`, `gammaNegG2`, `deltaNegG2` are stored
**pre-negated** — the values match SP1's `Groth16Verifier.sol`
`BETA_NEG_X_0`, `BETA_NEG_X_1`, ..., `DELTA_NEG_Y_1` constants byte-for-byte.
This matches Rúnar's witness-assisted verifier convention after the SP1
refactor: β, γ, δ live on the G2 side pre-negated, and the verifier checks
`e(A,B) · e(L,-γ) · e(C,-δ) · e(α,-β) = 1`.

Fp2 component order is `(x0 = real, x1 = imag)` — this matches SP1's
Solidity convention (`BETA_NEG_X_0` = real, `_X_1` = imaginary). **No swap
is required** between the Solidity constants and Rúnar's on-chain format.

## Regenerating the fixtures

```bash
# 1. Build the bsv-evm host-evm prover
cd /path/to/bsv-evm/prover/host-evm
cargo run --release

# 2. Copy the refreshed artifacts to this directory
cp /path/to/bsv-evm/tests/sp1/groth16_raw_proof.hex \
   /path/to/bsv-evm/tests/sp1/groth16_encoded_proof.hex \
   /path/to/bsv-evm/tests/sp1/groth16_vkey_hash.hex \
   /path/to/bsv-evm/tests/sp1/groth16_public_inputs.txt \
   /path/to/bsv-evm/tests/sp1/public_values.hex \
   /path/to/runar/tests/vectors/sp1/v6.0.0/

# 3. If SP1 upgraded, regenerate vk.json from the new
#    Groth16Verifier.sol constants (see ~/.sp1/circuits/groth16/v6.0.0/).
```
